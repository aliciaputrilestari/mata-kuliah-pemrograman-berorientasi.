01 - Pengertian OOP

OOP (Object Oriented Programming) adalah paradigma atau konsep pemrograman yang berorientasi pada objek, di mana program dibuat berdasarkan model objek yang memiliki data (atribut) dan perilaku (method/fungsi).

02 - KONSEP DASAR

Class (Kelas) → Cetakan atau blueprint untuk membuat objek.

Object (Objek) → Hasil dari class (wujud nyata dari kelas).

Attribute (Atribut) → Data atau sifat yang dimiliki objek.

Method → Fungsi atau perilaku yang dapat dilakukan objek..

03 - CONTOH CLASS SEDERHANA (CAR)

Di bawah ini adalah contoh pendefinisian class Car. Class ini memiliki dua attribute (model dan color) serta dua method (start dan stop) yang merepresentasikan perilaku dasar sebuah mobil.

// Nama   : Alicia Putri Lestari
// Nim    : 251410011
// Kelas  : Si2A

class Car {
  String? model;
  String? color;
  void start() {
    print('Car started!');
  }
  void stop() {
    print('Car stopped!');
  }
}
void main() {
  Car myCar = Car();
  
  myCar.model = "Brio";
  myCar.color = "Pink";
  myCar.start();
  myCar.stop();
}

04 - OBJECT INSTANTIATION

Setelah class didefinisikan, kita dapat membuat object atau instance dari class tersebut di dalam fungsi main(). Proses ini disebut instansiasi, di mana objek dialokasikan ke dalam memori.

// Nama   : Alicia Putri Lestari
// Nim    : 251410011
// Kelas  : Si2A

class Car {
  String? model; 
  String? color; 
}

void main() {
  Car myCar = Car();
  
  myCar.model = "Brio"; 
  myCar.color = "Pink";

  print(myCar.model); 
  print(myCar.color);
}

05 - CONSTRUCTORS

Constructor adalah metode inisialisasi yang dipanggil secara otomatis saat objek baru diciptakan dalam memori sistem. Di Dart, kita bisa menggunakan formal parameters (this.attribute) untuk menyederhanakan proses pengisian nilai attribute.

// Nama   : Alicia Putri Lestari
// Nim    : 251410011
// Kelas  : Si2A

class Car {
  String model; 
  String color;

  Car(this.model, this.color);

  void start() {
    print('Engine Online: $model'); 
  }
}

void main() {
  Car myCar = Car("Brio", "Pink");

  myCar.start(); 
  print(myCar.color); 
}

07 - ENCAPSULATION

Encapsulation adalah teknik untuk menyembunyikan detail implementasi internal. Di Dart, kita menggunakan awalan underscore (_) untuk membuat attribute bersifat private, sehingga hanya bisa diakses melalui getter dan setter.

// Nama   : Alicia Putri Leestari
// Nim    : 251410011
// Kelas  : Si2A

class Person {
  String _name; 
  int age;    

  Person(this._name, this.age);

  void setName(String name) => _name = name;
  
  String getName() => _name;
}

void main() {
  Person person = Person("Alicia Putri Lestari", 18);

  print(person.age); 
  person.setName("Alicia Putri Lestari");
  print(person.getName()); 
}

08 - THIS KEYWORD

Kata Kunci this digunakan untuk merujuk pada instance saat ini dari sebuah class. Ini sangat berguna untuk menghindari ambiguitas ketika nama parameter para constructor sama dengan nama attribute di dalam class.

// Nama   : Alicia Putri Lestari
// Nim    : 251410011
// Kelas  : Si2A

class Product {
  String nama; 
  Product(this.nama);
}

void main() {
  Product produk1 = Product("Laptop Ultrabook");
  print(produk1.nama); 
}










09 - PRACTICE 2: PRODUK CLASS


Latihan lanjutan menggunakan Named Parameters dan operator required. Teknik ini membuat pembuatan
objek menjadi lebih deskriptif dan aman karena compiler akan memberikan peringatan jika data 
wajib tidak diisi.




//Nama   : Alicia Putri Lestari
// Nim    : 251410011
// Kelas  : Si2A

class Produk {
  final String id, nama; 
  final double harga; 
  final int stok;   
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}

void main() {
  Produk laptop = Produk(
    id: "P001",
    nama: "Laptop Ulrabook",
    harga: 15000000,
    stok: 15,
    kategori: "Elektronik",
  );

  Produk smartphone = Produk(
    id: "P002",
    nama: "Smartphone Gaming",
    harga: 12000000,
    stok: 25,
  );

  Produk kamera = Produk(
    id: "P003",
    nama: "Kamera Action",
    harga: 20000000,
  );

  print(laptop.nama); 
  print(smartphone.nama); 
  print(kamera.nama); 
}












10- Kesimpulan


1. OOP (Object Oriented Programming)
OOP adalah konsep pemrograman yang menggunakan objek dan kelas untuk mengatur kode agar lebih terstruktur, mudah dipahami, dan mudah digunakan kembali.


2. Constructor
Constructor adalah method khusus dalam class yang otomatis dijalankan saat objek dibuat, biasanya digunakan untuk menginisialisasi nilai awal atribut.


3. Encapsulation
Encapsulation adalah konsep menyembunyikan data dalam class dan hanya bisa diakses melalui method tertentu, sehingga data lebih aman dan terkontrol.
